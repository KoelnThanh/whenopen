## Zusammenfassung

WhenOpen ist eine Android-App mit Widget, die persönliche Öffnungszeiten lokal speichert und auf einen Blick zeigt, welche gespeicherten Orte gerade geöffnet oder geschlossen sind. Die App löst das alltägliche Problem des wiederholten Nachschlagens von Öffnungszeiten fester, regelmäßig besuchter Orte — Behörden, Ärzte, Supermärkte, Freizeiteinrichtungen. WhenOpen ist kein Ersatz für Google Maps, sondern ein fokussierter persönlicher Zwischenspeicher: schnell befüllt, täglich nützlich, ohne Suche.

Technologiebasis: Flutter (Dart). Lokal, offline-fähig, kein Backend, keine Cloud.

- **Primärer Nutzer:** Einzelpersonen im Alltag — Eltern, Berufstätige, Pendler. Nicht zwingend technikaffin. Nutzen das Gerät allein.
- **Sekundäre Nutzer (Potenzial):** Sozialarbeiter, Selbstständige mit Kundenterminen — nutzen die App beruflich für eine überschaubare Menge fester Adressen.

## Kern-Workflows

### Workflow 1 — Neuen Eintrag anlegen (Schnelleintrag)

1. Nutzer öffnet WhenOpen
2. Tippt auf "Neuen Ort hinzufügen"
3. Gibt Name ein (Pflichtfeld)
4. Sequenzieller Dialog: Montag → Dienstag → … → Sonntag
   - Pro Tag: ein oder mehrere Zeitblöcke (von / bis) — „＋ weiterer Zeitblock" bildet Pausen ab (E9)
   - Zeiten des letzten geöffneten Tags als Vorschlag — Nutzer bestätigt oder passt an
   - Nicht ausgefüllte Tage = geschlossen (Default)
5. Kategorie wählen — vorhandene oder „＋ Neue Kategorie" (E15), optional
6. Optionale Felder: Adresse, Telefonnummer, Google Maps Link (URL)
7. Speichern → Eintrag erscheint in der Hauptliste
8. Hinweis bei 50 Einträgen ("Maximale Anzahl erreicht", E11)

**Abbruch / Validierung:**
- Name fehlt → Speichern nicht möglich, Hinweis anzeigen
- Kein einziger Tag mit Öffnungszeit eingetragen → Speichern nicht möglich, Hinweis anzeigen
- Nutzer bricht ab → kein Eintrag gespeichert, kein Dialog

### Workflow 2 — Tagesübersicht im Widget

1. Nutzer entsperrt Handy, wischt auf den dedizierten Widget-Screen
2. Widget zeigt alle Einträge für den aktuellen Tag als Liste:
   - Oben: aktuell geöffnete Orte (Name + aktuelle Öffnungszeit)
   - Unten: aktuell geschlossene Orte (Name + nächste Öffnungszeit oder "heute geschlossen")
3. Tippen auf einen Eintrag im Widget → App öffnet direkt die Detailansicht dieses Eintrags

### Workflow 3 — Hauptliste und Navigation

1. Nutzer öffnet WhenOpen
2. Sieht Liste aller gespeicherten Orte (alphabetisch sortiert)
3. Jeder Listeneintrag zeigt: Name + aktueller Status (offen / geschlossen)
4. Tippen auf Eintrag → Detailansicht
5. FAB (Floating Action Button) unten rechts → Schnelleintrag starten

### Workflow 4 — Detailansicht eines Eintrags

1. Nutzer tippt auf Eintrag in der Liste oder im Widget
2. Sieht Wochenübersicht (Mo–So) mit allen Öffnungszeiten inkl. Mittagspausen
3. Sieht optionale Felder: Adresse, Telefonnummer, Google Maps Link
4. Aktionen:
   - Eintrag bearbeiten (öffnet Bearbeitungsformular)
   - Google Maps öffnen (Link, springt in Google Maps App)
   - Eintrag löschen (mit „Rückgängig"-SnackBar ~5 s, dann endgültig — E13)

### Workflow 5 — Import-Assistent (OSM-Lookup)

**Entscheidung E2 (2026-06-01):** OSM/Nominatim als MVP-Import-Weg. Google Places API ist ToS-inkompatibel (Caching verboten). Share-Intent liefert keine Öffnungszeiten.

**Ablauf:**
1. Nutzer startet Schnelleintrag, tippt Ortsname in Suchfeld
2. App fragt Nominatim API ab (OpenStreetMap, kostenlos, Caching erlaubt)
3. Treffer werden als Liste angezeigt (Name, Adresse)
4. Nutzer wählt Treffer → verfügbare Felder (Name, Adresse, Telefon, Öffnungszeiten) werden vorbelegt
5. Nutzer prüft und bestätigt jedes Feld — Änderungen jederzeit möglich
6. Speichern wie bei manuellem Eintrag

**Wichtige Einschränkung:** Datenqualität OSM ca. 30% bei Supermärkten, bei kleinen Betrieben deutlich weniger. Der Import-Assistent ist ein optionaler Komfort-Helfer, kein verlässlicher Datenprovider. Der Nutzer muss immer bestätigen.

**Architekturprinzip (unverändert):**
Das Import-Modul bleibt eine austauschbare Komponente. Die App funktioniert vollständig ohne Import. Bei fehlgeschlagenem Lookup oder fehlenden Öffnungszeiten: leeres Schnelleintrag-Formular öffnen.

**Für Version 2 (nach MVP):** Share-Intent als UX-Eingang + rechtliche Klärung zu alternativen Datenquellen. HERE Maps (250K Free Requests/Monat) als mögliche Alternative zu evaluieren.

## Datenmodell

### Entität: Ort (Location)

| Feld | Typ | Pflicht | Beschreibung |
|---|---|---|---|
| `id` | UUID | ja | Interne ID, automatisch generiert |
| `name` | String | ja | Anzeigename des Orts |
| `oeffnungszeiten` | Array[Tag] | ja | 7 Einträge (Mo–So), mind. 1 mit Zeiten |
| `adresse` | String | nein | Straße, PLZ, Ort |
| `telefon` | String | nein | Telefonnummer |
| `google_maps_link` | String (URL) | nein | Direktlink zum Google Maps Eintrag |
| `kategorie` | UUID (Kategorie-ID) | nein | Referenz auf eine Kategorie (E15). Leer → Gruppe „Sonstige" |
| `erstellt_am` | ISO-8601 Datum | ja | Automatisch gesetzt |
| `geaendert_am` | ISO-8601 Datum | ja | Automatisch aktualisiert |

### Entität: Tag (OpeningDay) — **E9: Zeitblöcke statt einer Pause**

| Feld | Typ | Pflicht | Beschreibung |
|---|---|---|---|
| `wochentag` | Enum (mo/di/mi/do/fr/sa/so) | ja | Wochentag |
| `geoeffnet` | Boolean | ja | abgeleitet: `zeiten` ist nicht leer |
| `zeiten` | Array[{`von`,`bis`}] | ja | Liste von Zeitblöcken (HH:MM). Leer = geschlossen. Eine Pause = Lücke zwischen zwei Blöcken; mehrere Pausen = mehrere Blöcke. |

*`pause_von`/`pause_bis` entfallen (E9) — sie sind als Lücke zwischen Blöcken modelliert.*

### Entität: Kategorie (E15)

| Feld | Typ | Pflicht | Beschreibung |
|---|---|---|---|
| `id` | UUID | ja | Stabil; Einträge & Widgets verweisen darauf |
| `name` | String | ja | Anzeigename, frei umbenennbar |
| `farbe` | String (Hex) | nein | Wiedererkennung in Liste/Widget |
| `sortierung` | Int | ja | Reihenfolge in Liste/Widget |

*„Sonstige" ist eine implizite, nicht löschbare Auffang-Kategorie (`kategorie == null`).*

### Zustandslogik "Jetzt geöffnet?" — **E9**

- Geöffnet ⇔ aktuelle Uhrzeit liegt in **irgendeinem** Block (`von`–`bis`) des aktuellen Wochentags
- „schließt um" = `bis` des aktuellen Blocks; Zeit zwischen zwei Blöcken (Pause) → geschlossen
- Leere `zeiten` / kein Eintrag für den Tag → geschlossen
- Keine Feiertags-Logik im MVP (reguläre Wochentagslogik gilt immer)
- Aktualisierung von App & Widget: siehe E16 (`1.3`)

### Speicherung

- **Format:** JSON-Datei, lokal auf dem Gerät
- **Pfad:** App-internes Verzeichnis (nicht öffentlich zugänglich ohne Export)
- **Export:** Share-Intent — Nutzer kann die JSON-Datei teilen (z.B. per E-Mail, WhatsApp)
- **Limit:** 50 Einträge (E11 — kommuniziert, nicht technisch blockiert)
- **Fehlerbehandlung:** Bei korrupter JSON-Datei → Backup anlegen (`whenopen_backup_[timestamp].json`), leere App starten, Nutzer per Dialog informieren

## Integrationen

- **OSM/Nominatim (Import-Assistent):** Kostenlose OpenStreetMap-API, Caching erlaubt (ODbL-Lizenz). Datenqualität ~30% bei Supermärkten. Nutzer bestätigt alle Daten vor dem Speichern. Modul ist austauschbar. Fallback: manueller Eintrag funktioniert immer.
- **Google Places API:** Ausgeschlossen — ToS verbietet lokales Caching von Öffnungszeiten. Fundamentaler Konflikt mit WhenOpen-Konzept (Research-Ergebnis 2026-06-01).
- **Google Maps (Aufruf aus App):** Direktlink öffnet Google Maps App via URL (`https://maps.google.com/?cid=...`). Kein SDK, kein API-Key erforderlich.
- **Weitere externe Systeme:** keine
- **Auth gegenüber externen Systemen:** keine (kein Login, keine API-Keys im MVP)
- **Netzwerk:** nur für optionalen Import benötigt; Kernfunktion vollständig offline

## Edge Cases und Fehlerfälle

| Situation | Verhalten |
|---|---|
| Name fehlt beim Speichern | Speichern blockiert, Hinweistext unter dem Feld |

| Tag nicht eingetragen / geschlossen | Ort gilt als geschlossen für diesen Tag |
| 50 Einträge erreicht | Hinweistext: "Maximale Anzahl von 50 Einträgen erreicht" — kein harter Block (E11) |
| JSON-Datei korrupt | Backup anlegen, leere App starten, Nutzer per Dialog informieren |
| Google Maps App nicht installiert | Google Maps URL im Browser öffnen als Fallback |
